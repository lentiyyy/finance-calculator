## Обзор проекта

Веб‑приложение для учёта **доходов и расходов** на чистом **HTML + CSS + JavaScript** (без фреймворков).  
Данные сохраняются локально через **localStorage** и не отправляются на сервер.

Функции:
- добавление транзакций (доход/расход) с суммой, категорией, датой и комментарием;
- фильтрация по периоду / типу / категории и поиск по комментарию;
- таблица транзакций с возможностью редактирования и удаления;
- сводка: итоговые доходы, расходы и баланс;
- аналитика: распределение расходов по категориям **в рублях и процентах**;
- текстовый график из символов `#`;
- бюджеты по категориям расходов с отображением остатка или превышения;
- экспорт текущей выборки транзакций в CSV;
- загрузка демонстрационных данных по кнопке;
- очистка всех транзакций с подтверждением;
- валидация и уведомления об ошибках через toast;
- адаптивная вёрстка.

## Запуск

Так как используются ES‑модули (`type="module"`), нужен локальный HTTP/HTTPS.

### Вариант 1: Python
```bash
cd app
python3 -m http.server 8000
```
Открыть: `http://localhost:8000`

### Вариант 2: Node.js (serve)
```bash
cd app
npx serve .
```

## Данные в localStorage

Ключи хранения:
- `finance-calc:transactions:v1`
- `finance-calc:budgets:v1`

В `localStorage` хранится **массив объектов** транзакций, например:
```json
{
  "id": "abc-123",
  "type": "expense",
  "amount": 1250.5,
  "category": "Еда",
  "date": "2026-05-14",
  "comment": "продукты"
}
```

При пустом `localStorage` приложение открывается с пустым списком. Для демонстрации можно нажать кнопку **«Загрузить демо-данные»**: тестовые транзакции создаются только до текущей даты.
